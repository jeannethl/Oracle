******************************************************
		DESCONFIGURAR UN NODO DEL CLUSTER 
******************************************************

1.- Verificar desde nodo plbdcb01:

Con usuario root:

cd /oracle/app/product/19c/grid/bin
./olsnodes -s -t


2.- Desde el nodo plbdcb02 el cual se desea desincorporar del cluster:

	2.1.-Desinstalar el binario 

A.- Loguearse con usuario grid

##documentar los pasos si estan compartidos el binario

B.- Ir al directorio de oracle home de grid y ejecutar

cd $ORACLE_HOME
$ORACLE_HOME/deinstall/deinstall -local 


En caso de identificar el directorio TEMP:

$ORACLE_HOME/deinstall/deinstall -local -tmpdir <path to tmp location>


C.- Seguir los pasos leer detenidamente


3.- Desde un nodo plbdcb01 ejecutar los pasos para eliminar nodo

	3.1.- Eliminar nodo del cluster

crsctl delete node -n node_to_be_deleted
crsctl delete node -n plbdcb02


	3.2.- Ejecutar el cluvfy

Con usuario grid:

cluvfy stage -post nodedel -n plbdcb02 -verbose


	3.3.- Consultar la vip del nodo 

srvctl config vip -node plbdcb02


	3.4.- Detener vip 

srvctl stop vip -vip plbdcb02-vip.banvenez.corp


	3.5.- Remove vip 

con root usuario:

srvctl remove vip -vip plbdcb02-vip.banvenez.corp

Please confirm that you intend to remove the VIPs plbdcb02-vip.banvenez.corp (y/[n]) y


4.- Desde el nodo plbdcb02 desincorporado limpiar el directorio /oracle 




********************************************************************************************
					AGREGAR UN NODO DEL CLUSTER GRID_HOME
*******************************************************************************************


En el nodo plbdcb02 el cual se desea reincorporar en el cluster ejecutar:

1.- Crear los directorios 

mkdir -p /oracle/app/oraInventory
mkdir -p /oracle/app/grid		            
mkdir -p /oracle/app/product/19c/grid
mkdir -p /oracle/app/oracle/product/19c/db1

chown -R grid:oinstall /oracle/
chown -R oracle19:oinstall /oracle/app/oracle
chmod -R 775 /oracle/


2.- Verificar si todo esta corectamente para agregar el nodo

Desde el nodo master plbdcb01 ejecutar:

cluvfy comp peer -refnode plbdcb01 -n plbdcb02 -verbose

cluvfy stage -pre nodeadd -n plbdcb02 -fixup -verbose


NOTA: Como buena practica todas las revisiones deben terminar con estatus PASS

RELACION DE CONFIANZA

3.- Verificar y Configurar la relación de confianza entre los nodos 

	Configuración de relación de confianza

Realizar los siguientes pasos con el usuario oracle19, grid y root en todos los nodos del clúster.

3.1.-Generar las llaves públicas

En todos los nodos se ejecuta:

cd 
mkdir .ssh
chmod 700 .ssh
cd .ssh
ssh-keygen -t rsa
ssh-keygen -t dsa

3.2.-Se copia el contenido de las llaves publicas al archivo authorized_keys.$(hostname) 

cat *.pub >> authorized_keys.$(hostname)


3.3.-Se envía el archivo authorized_keys.$(hostname) a los otros nodo con el que se realizará la relación de confianza

Ejemplo:

Nodo 1 a nodo 2
scp authorized_keys.$(hostname) user@Nodo2:~/.ssh/

Nodo 2 a nodo 1
scp authorized_keys.$(hostname) user@Nodo1:~/.ssh/


3.4.Copiar el contenido de authorized_keys.$(hostname) al archivo authorized_keys

En todos los nodos Ejecutar:

cd ~/.ssh
cat authorized_keys.* >> authorized_keys
chmod 600 authorized_keys

3.5.-Ejecutar las pruebas 

ssh Nodo#
